# Документация по дизайну и DesignOps

Эта папка — общий файловый источник требований по дизайну сайта `дом-эксперт.рф` для работы через ChatGPT Web, opencode и WP-CLI/SSH.

## Назначение

Документы нужны, чтобы не передавать промты вручную из ChatGPT Web в opencode. ChatGPT Web уточняет и оформляет требования, opencode читает файлы из репозитория и выполняет задачи через доступ к сайту.

## Основные файлы

| Файл | Назначение |
|---|---|
| `01-дизайн-контекст-и-правила-агента.md` | Основной дизайн-контекст, визуальная концепция, токены, ограничения, правила для агента |
| `02-инструкция-дизайн-ревью.md` | Как проверять сайт после изменений, формат отчёта, viewport, severity |
| `03-opencode-designops-workflow.md` | Как opencode должен работать с документацией, задачами и проверками |
| `04-карта-ассетов.md` | Какие flat-ассеты использовать в WordPress и где они применяются |

## Где фиксировать новые замечания

Все замечания после проверки дизайна фиксируются не в этой папке, а в:

```text
/беклог-промтов/
```

Там лежат атомарные задачи для opencode. Каждая задача должна быть проверяемой: один блок, один экран или одна проблема.

## Важное правило

Структура сайта, URL и меню уже есть в рабочем контексте opencode. В этой папке структура не дублируется как главный источник истины. Здесь фиксируются дизайн-правила, UX/UI-проверки, ограничения и регламент работы.

## Базовая техническая связка

```text
WordPress
Kadence Theme
Kadence Blocks
Gutenberg
Safe SVG
WP-CLI / SSH через opencode
Собственный AI/WordPress-плагин проекта для CPT/полей, не ACF
```

## Принцип работы

```text
проверка дизайна → замечание → файл в /беклог-промтов/ → opencode выполняет → скриншоты/отчёт → подтверждение пользователя
```

Не переходить к следующему крупному этапу без подтверждения результата пользователем.
