# Карта применения ассетов

Используется flat asset pack без папок. Имена файлов имеют формат:

```text
NN-category-name.ext
```

## Важно для WordPress Media Library

В медиабиблиотеке WordPress файлы хранятся **плоско, без учёта папок**. Поэтому в Gutenberg/Kadence и при поиске через WP-CLI нужно ориентироваться на:

- имя файла;
- title вложения;
- attachment ID;
- URL файла.

Папки из репозитория `/документация/assets/` нужны только для удобства исходников в Git. На production WordPress они не существуют как структура медиатеки.

Фактическое состояние медиабиблиотеки фиксируется отдельно:

```text
/документация/05-prod-media-map.md
```

## Группы

```text
00 — documentation
01 — brand
02 — navigation
03 — path / сценарии и инструменты
04 — property / параметры объекта
05 — status
06 — trust
07 — form
08 — risk
09 — poster
10 — favicon
```

## Дополнительно видно в медиабиблиотеке

```text
favicon.svg
cropped-10-favicon-favicon.jpg
```

`cropped-10-favicon-favicon.jpg` — производный файл WordPress после обрезки иконки. Не считать исходным asset pack.

## Правила использования

- Для SVG должен быть активен Safe SVG.
- Risk-файлы не использовать без текстового описания уровня риска.
- SVG-постеры использовать как изображения или фоновые медиа в Kadence Row Layout / Gutenberg Cover.
- В WordPress Media Library нет папок: искать по flat-имени файла.
- Если ассет не найден в медиатеке, opencode должен проверить `/документация/assets/`, затем подготовить импорт через `wp media import`, но не выполнять без подтверждения пользователя.

## Что не делать

- Не использовать generic stock вместо проектных poster-файлов, если нужный poster есть в asset pack.
- Не использовать risk-иконку как единственное обозначение риска.
- Не менять логотипы на произвольные версии без подтверждения.
- Не подменять целевой visual случайной недвижимостью из интернета.
