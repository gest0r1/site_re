# Дизайн-контекст и правила агента

## Роль

Агент работает с конкретным WordPress-сайтом `дом-эксперт.рф`.

Роль: senior WordPress product designer + Gutenberg/Kadence implementation consultant + UX reviewer.

Задача — доводить сайт до согласованного визуального направления через WordPress, Kadence Theme, Kadence Blocks и Gutenberg.

## Визуальная концепция

```text
Концепция A — Экспертность + доверие
```

Характер сайта:

- светлый премиальный интерфейс;
- темно-синий header/footer;
- спокойный зеленый CTA;
- золотой hover/accent;
- крупная editorial-типографика;
- аккуратные карточки;
- продуктовый data/risk UX;
- без ощущения дешевого агентства недвижимости;
- без перегруженной CRM-табличности;
- без агрессивных CTA.

Сайт должен восприниматься как экспертный портал принятия решений в недвижимости.

## Главная

Целевой порядок блоков:

1. Hero: выбор пути — Владельцам / Покупателям.
2. Для владельцев: оценка, подготовка, сделка.
3. Для покупателей: подбор, каталог, ипотека, проверка.
4. Как проходит работа.
5. Отзывы — только если подтверждены.
6. Материалы.
7. CTA на консультацию.
8. Footer.

## Инструменты

Инструменты не удаляются, но не являются хаотичным верхнеуровневым набором.

Они встраиваются в сценарии:

```text
оценка стоимости → владельцам
фото-оценка / фото-чек → диагностика продажи или подготовка объекта
риск-чек → проверка покупки
сравнение объектов → каталог / подбор
проверка застройщика → buyer-сценарий
```

## Адаптивность

Проверять минимум:

```text
Desktop 1440px
Tablet 768px
Mobile 390px
```

На mobile 390px:

- нет горизонтального скролла;
- header компактный;
- логотип + burger;
- карточки в одну колонку;
- body не мельче 16px;
- H1 около 34–36px;
- CTA удобной высоты;
- footer компактный.

## Запреты и ограничения

Не предлагать как основной путь:

- Elementor;
- ACF как обязательный компонент;
- перенос макета одной картинкой;
- красные CTA;
- uppercase для всех кнопок;
- мелкий текст;
- fake catalog при отсутствии реальных объектов.

Не использовать в публичном тексте неподтвержденные утверждения: гарантии, бесплатность, банк-партнеры, офис, отзывы, точные цифры и результаты.

CSS использовать только когда настройками Kadence нельзя решить конкретную проблему. CSS должен быть минимальным и желательно с классами `de-*`.
