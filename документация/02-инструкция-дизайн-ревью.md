# Инструкция дизайн-ревью

## Роль ревьювера

Ревьювер — независимый UX/UI QA reviewer для сайта `дом-эксперт.рф`.

Задача — сравнивать фактическое состояние WordPress/Kadence/Gutenberg-сайта с целевым дизайном `Концепция A — Экспертность + доверие`.

Ревьювер не придумывает новый дизайн и не оценивает «нравится / не нравится». Он фиксирует отклонения от целевого состояния.

## Что нужно для ревью

Перед ревью нужны:

1. URL сайта.
2. Страница или блок, который проверяется.
3. Скриншоты после настройки, если live-доступ ограничен.
4. Целевые макеты, если доступны.
5. Уточнение этапа: header, footer, главная, mobile, каталог, формы, весь сайт.

Если ревью основано только на скриншотах, явно писать:

```text
Ревью основано только на предоставленных скриншотах.
```

## Viewports

Проверять минимум:

```text
Desktop: 1440px
Tablet: 768px
Mobile: 390px
```

Если возможно, дополнительно:

```text
Desktop wide: 1920px
Mobile small: 360px
```

На каждом viewport проверить:

- горизонтальный скролл;
- высоту header;
- читаемость;
- карточки;
- CTA;
- footer;
- отсутствие наложений.

## Severity

```text
P0 — блокирует использование / сайт сломан
P1 — сильно ломает дизайн или конверсию
P2 — заметное отклонение от дизайна
P3 — косметика
```

## Формат отчета

Всегда выдавать таблицу:

| ID | Severity | Экран/URL | Viewport | Что не так | Как должно быть | Рекомендация |
|---|---|---|---|---|---|---|

После таблицы дать:

```text
Критичные исправления следующего шага:
1. ...
2. ...
3. ...
```

Не давать 30 рекомендаций сразу. Фокус — 3–7 наиболее важных.

## Header desktop

Ожидание:

```text
[logo] [menu] [green CTA]
```

Проверить:

| Критерий | Ожидание |
|---|---|
| Header background | #10233F |
| Logo | светлый, видимый, не размытый |
| Menu | белое, hover gold #C8A468 |
| CTA | зеленая кнопка справа |
| Height | около 72–80px |
| Dropdown | темный, читаемый |

Критичные ошибки:

- sticky header становится белым;
- меню пропадает при прокрутке;
- dropdown белый с белым текстом;
- логотип слишком крупный;
- меню не помещается;
- CTA прижат к краю.

## Header mobile / sticky

Mobile:

```text
[logo] [burger]
height около 64px
```

Проверить:

- нет горизонтального скролла;
- burger видим;
- меню открывается;
- dropdown не выходит за экран;
- sticky не закрывает слишком много контента;
- sticky-сжатие не ломает логотип.

## Footer

Цель — компактный темный footer.

Проверить:

| Критерий | Ожидание |
|---|---|
| Background | #10233F |
| Content max width | 1180–1240px |
| Desktop columns | 4 компактные колонки |
| Height desktop | примерно 190–230px |
| Links | без bullet-маркеров |
| Headings | белые |
| Text | светло-серый |
| Hover | золотой |
| Mobile | stack или accordion |

Ошибки:

- footer растянут на всю ширину с огромными пустотами;
- заголовки черные на темном фоне;
- списки с bullet-маркерами;
- слишком большая высота;
- юридическая строка плохо читается;
- mobile footer слишком длинный без структуры.

## Главная страница

Проверить порядок блоков:

1. Hero: выбор пути — Владельцам / Покупателям.
2. Для владельцев.
3. Для покупателей.
4. Как проходит работа.
5. Отзывы — только подтвержденные.
6. Материалы.
7. CTA.
8. Footer.

Hero:

| Критерий | Ожидание |
|---|---|
| H1 | крупный editorial |
| Value text | ясно объясняет пользу |
| CTA | оценка / подбор / консультация |
| Visual | real-estate poster, не generic stock |
| First screen | понятно, что сайт про экспертную недвижимость и два пути |

Ошибки:

- стандартный заголовок «Главная» занимает верх;
- hero слишком низко;
- карточки не выровнены;
- CTA слабые;
- слишком много пустого пространства;
- нет очевидного выбора Владельцам / Покупателям;
- инструменты вынесены хаотичным набором ссылок;
- используются неподтвержденные обещания.

## Формы и инструменты

Проверять:

- форма понятна;
- labels видны;
- placeholder не заменяет label;
- есть success state;
- есть следующий шаг;
- есть дисклеймер;
- есть consent/privacy checkbox или связанный legal/consent текст;
- mobile ввод удобен.

Ошибки:

- форма требует телефон до любой пользы;
- нет объяснения, что пользователь получит;
- нет success state;
- нет privacy/disclaimer/consent;
- поля слишком мелкие на mobile;
- инструмент выглядит отдельно от сценария владельца или покупателя.

## Минимальный чеклист финальной приемки

- [ ] Header desktop соответствует концепции.
- [ ] Header sticky не белеет и не теряет меню.
- [ ] Header mobile не ломается.
- [ ] Footer компактный, не растянутый.
- [ ] Главная имеет понятный hero.
- [ ] Путь Владельцам / Покупателям виден сразу или почти сразу.
- [ ] Инструменты встроены в крупные сценарии.
- [ ] CTA заметны, но не агрессивны.
- [ ] Риски имеют текстовое объяснение.
- [ ] Нет горизонтального скролла на 390px.
- [ ] Основной текст не мельче 16px.
- [ ] Нет generic stock-photo ощущения.
- [ ] Нет дешевого агентства недвижимости.
- [ ] Нет перегруженной CRM-табличности.
