# Release Principles

## Назначение
Единый release gate для всех этапов проекта.

## Правило
Каждый этап можно выкатывать в прод только если он самодостаточен и проверен.

## Checklist
- [ ] backup создан
- [ ] restore-check выполнен
- [ ] smoke tests пройдены
- [ ] rollback path подтвержден
- [ ] approval owner назначен
- [ ] release notes готовы
- [ ] deploy выполнен
- [ ] post-deploy check пройден

## Что проверять после релиза
- главная
- формы
- меню
- медиа
- ключевые шаблоны
- лог ошибок

## Fail policy
Если любой пункт не пройден, релиз не идёт.
