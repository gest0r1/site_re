# Этап 7. Первичное наполнение и AI workflow

## Зачем
Создать живой сайт, а не только оболочку.

## Что делает OpenCode
- структура страниц
- стартовые тексты
- черновики разделов
- служебные шаблоны

## Что делает WordPress AI потом
- новые статьи
- новости
- правки текстов
- быстрые обновления контента

## Editorial workflow
draft → AI generation → human review → SEO check → factual/legal check → publish

## Rules
- no personal data in AI prompts by default
- client approval required for public publication
- tone/style guide fixed once
- duplicate / plagiarism check before publish

## AI publication gate
- [ ] no unsupported legal/financial guarantees
- [ ] real estate claims have source or reviewer approval
- [ ] no personal data in prompt/output
- [ ] plagiarism check passed
- [ ] client approval recorded

## Owner
- **Owner:** content editor + client

## Что наполняем
- главная
- о компании
- услуги
- статьи
- новости
- отзывы
- FAQ

## Тесты этапа
- [ ] all key pages accessible
- [ ] texts are not empty
- [ ] CTA links go to right place
- [ ] editor can update content safely
- [ ] draft-to-publish workflow works
- [ ] AI output reviewed before publish
- [ ] AI draft gets blocked until human approval is recorded

## Release gate
После заполнения и проверки контента — прод.
