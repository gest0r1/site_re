# Инструкция для ИИ: SVG → Gutenberg

## Алгоритм

1. Классифицируй блок: navigation, paths, property, status, risk, trust, forms, states, posters, decor или gutenberg.
2. Найди точный `asset_id` в `ASSET-CATALOG.md`.
3. Проверь `recommended_block`, `usage`, `currentColor`, `accessibility`.
4. Вставь ресурс рекомендуемым способом.
5. Добавь текстовый эквивалент, если SVG информативный.
6. Проверь 390, 768 и 1440 px.

## Особые правила

- Property card: не более 4 характеристик в каталоге.
- Risk: название + уровень + причина + действие; цвет не является единственным кодом.
- Forms: иконка не заменяет label.
- States: один SVG, заголовок, пояснение, одно действие.
- Posters: desktop/mobile не взаимозаменяемы; placeholder явно обозначается.
- Decor: только фон, всегда `aria-hidden`.
- Gutenberg preview: никогда не публиковать на фронтенде.

## Проверка перед завершением

- [ ] asset_id существует
- [ ] выбран по смыслу
- [ ] usage соблюден
- [ ] a11y соблюдена
- [ ] risk имеет текст
- [ ] mobile poster выбран отдельно
- [ ] нет горизонтального скролла
- [ ] нет абсолютной раскладки всей страницы
