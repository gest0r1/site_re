# Беклог промтов

Эта папка нужна для фиксации замечаний после проверки дизайна и передачи их в opencode как атомарных задач.

## Зачем это нужно

Проверка дизайна у opencode может быть ограничена: не всегда он надежно видит визуальные отклонения, sticky-состояния, mobile-поведение, spacing, визуальную иерархию и соответствие макету.

Поэтому замечания после ручной проверки или проверки в ChatGPT Web фиксируются здесь как отдельные markdown-задачи.

## Правило

Один файл — одна проверяемая задача.

Плохо:

```text
Исправить главную.
```

Хорошо:

```text
003-home-hero-spacing-mobile.md
004-header-sticky-white-background.md
005-footer-too-tall-desktop.md
```

## Как именовать файлы

```text
NNN-area-problem.md
```

Примеры:

```text
001-sync-design-context.md
002-header-sticky-color.md
003-footer-compactness.md
004-home-hero-mobile.md
005-seller-cards-alignment.md
```

## Статусы внутри файла

```text
new
in-progress
done
rejected
needs-user-check
```

## Как opencode должен выполнять задачу

Команда:

```text
Прочитай /документация/README.md и выполни /беклог-промтов/NNN-name.md. Структуру сайта бери из своего текущего контекста. Не переходи к следующей задаче без подтверждения пользователя.
```

## После выполнения

Opencode должен дописать в конец файла:

- что изменено;
- какие настройки/файлы затронуты;
- какие URL проверены;
- какие viewport проверены;
- какие скриншоты сделаны;
- что осталось проверить пользователю.

## Когда создавать новую задачу

Создавать новую задачу, если:

- пользователь увидел визуальное отклонение;
- ChatGPT Web по скриншоту нашел проблему;
- opencode сообщил, что не может проверить визуально;
- после исправления появилась новая проблема;
- нужно провести отдельное ревью блока.

## Когда не создавать задачу

Не создавать задачу для общих обсуждений и решений. Общие правила фиксируются в `/документация/`.
