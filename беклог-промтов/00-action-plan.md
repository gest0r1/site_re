# План анализа и доработки остальных страниц — poster version

## Что изменено относительно предыдущего пакета

Предыдущая версия для страниц `109–140` содержала технические SVG-постеры. По замечанию пользователя этот подход признан некорректным для production visual.

Новая версия:

- использует **растровые WebP poster-assets**;
- строится по тому же принципу, что и главная;
- сохраняет отдельный task-файл на каждую страницу;
- использует общий пул из 10 качественных poster-файлов, переиспользуемых по семействам страниц;
- исключает старые SVG-постеры из production-сценария.

## Порядок работы OpenCode

1. Открыть страницу.
2. Зафиксировать фактические блоки production.
3. Сверить назначение страницы с Gate 2 v2.
4. Взять poster-файл из `беклог-промтов/assets/site-pages/`.
5. Внедрить hero visual и при необходимости supporting visual.
6. Проверить 1440 / 768 / 390.
7. Заполнить отчёт в task-файле.


## Очистка версии v3

- удалены все SVG-файлы с назначением `poster`;
- удалены производные старых SVG страниц `109–140`;
- hero главной переведён на отдельные WebP desktop/mobile;
- SVG сохранены только для логотипов, иконок и wireframe-документации;
- task-файлы очищены от ссылок на удалённые poster SVG.
