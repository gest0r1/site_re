# 119 — Каталог или подбор объектов

## Статус

```text
new
```

## Источник замечания

```text
Gate 2 v2 / дизайн-система / reviewer instructions
```

## Проблема

Для страницы или шаблона «Каталог или подбор объектов» требуется согласованный WebP-постер и проверяемая инструкция внедрения. Старые многослойные SVG-постеры использовать нельзя.

## Где

```text
URL/шаблон: /buyers/catalog/
Блок: hero и ключевые смысловые секции
Viewport: 1440px / 768px / 390px
```

## Назначение

каталог при наличии реальных объектов либо честный lead-capture подбор

## Как должно быть

Использовать production WebP:

```text
беклог-промтов/assets/site-pages/poster-catalog-exterior-v1.webp
```

Постер используется в hero/intro и не подменяет фотографии конкретного объекта.

Размещение:

1. На desktop — текст и CTA слева, постер справа.
2. На tablet — две колонки либо аккуратный stack.
3. На mobile — текст, CTA, затем изображение; без горизонтального скролла.
4. `object-fit: cover`, скругление 18–24px, без текста и CTA внутри изображения.
5. Перед изменениями OpenCode обязан проверить фактическую production-страницу.

## Что можно менять

- [x] Gutenberg/Kadence Blocks;
- [x] CSS;
- [x] контент страницы;
- [x] медиа/ассеты;
- [x] шаблон/CPT собственного плагина, если применимо.

## Что нельзя менять

- Не менять URL и меню без подтверждения.
- Не добавлять Elementor или обязательный ACF.
- Не использовать удалённые SVG-постеры.
- Не создавать fake catalog и не выдавать постер за фото реального объекта.

## Критерии приемки

- [ ] Использован указанный WebP-файл.
- [ ] Нет ссылок на старые poster SVG.
- [ ] Проверены 1440px, 768px и 390px.
- [ ] На 390px нет горизонтального скролла и обрезанных CTA.
- [ ] Изображение не искажено и не выдано за реальный объект.
- [ ] Не сломаны header, sticky header, соседние блоки и footer.
- [ ] Alt заполнен либо осознанно оставлен пустым для декоративного изображения.
- [ ] Отчёт OpenCode заполнен.
