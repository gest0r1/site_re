# 000 — Шаблон задачи для opencode

## Статус

```text
new
```

## Источник замечания

```text
пользователь / ChatGPT Web / opencode reviewer / скриншот / live-проверка
```

## Проблема

Кратко описать, что не соответствует дизайну.

## Где

```text
URL:
Блок:
Viewport:
Устройство/браузер, если важно:
```

## Как сейчас

Фактическое состояние. Если вывод основан на скриншоте, указать:

```text
Вывод основан на предоставленном скриншоте.
```

## Как должно быть

Конкретный ожидаемый результат по дизайн-документации.

## Что можно менять

Отметить применимое:

- [ ] настройки Kadence Customizer;
- [ ] Gutenberg/Kadence Blocks;
- [ ] CSS;
- [ ] шаблон темы;
- [ ] контент страницы;
- [ ] медиа/ассеты;
- [ ] другое.

## Что нельзя менять

- Не менять структуру сайта и URL без подтверждения.
- Не менять header/footer menu без подтверждения.
- Не добавлять Elementor.
- Не добавлять ACF как обязательный компонент.
- Не использовать неподтвержденные факты.
- Не ломать mobile 390px.

## Критерии приемки

- [ ] Исправление видно на нужном viewport.
- [ ] Нет горизонтального скролла на mobile 390px.
- [ ] Не сломаны соседние блоки.
- [ ] Сохранена концепция A: navy / warm / green CTA / gold hover.
- [ ] Opencode указал, что именно изменено.
- [ ] Пользователь подтвердил визуально, если автоматическая проверка ограничена.

## Команда для opencode

```text
Прочитай /документация/README.md и выполни эту задачу. Перед изменениями прочитай дизайн-контекст и инструкцию ревью. Структуру сайта бери из своего текущего контекста. После выполнения допиши отчет в этот файл.
```

## Отчет opencode

Заполняется после выполнения.

```text
Дата:
Что изменено:
Затронутые файлы/настройки:
Проверенные URL:
Проверенные viewport:
Скриншоты:
Оставшиеся риски:
Нужно подтверждение пользователя: да/нет
```
