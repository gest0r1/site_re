# 102 — Добавить P1-постер владельцев на главную

## Статус

```text
new
```

## Источник замечания

```text
ChatGPT Web / проектный дизайн / макет главной
```

## Проблема

На главной странице блок для владельцев не имеет полноценного предметного визуала, который поддерживает концепцию «Экспертность + доверие» и отделяет сценарий продажи от остальных разделов.

## Где

```text
URL: /
Блок: «Для владельцев» / сценарий продажи недвижимости
Viewport: desktop 1440px, tablet 768px, mobile 390px
Устройство/браузер, если важно: все актуальные браузеры
```

## Как сейчас

В блоке используется только текст, иконка либо недостаточно выразительный графический элемент. Отдельного фотореалистичного постера для сценария владельца нет.

Вывод основан на проектной документации и целевом макете. Точное текущее состояние продакшена перед изменением проверить отдельно.

## Как должно быть

Использовать файл:

```text
беклог-промтов/assets/home-sellers-poster-v1.webp
```

Назначение изображения:

- сценарий владельца недвижимости;
- оценка, подготовка объекта и продажа;
- визуально спокойная премиальная недвижимость;
- без текста, логотипов и интерфейсных элементов внутри изображения.

Рекомендуемый alt:

```text
Подготовка квартиры к продаже и оценке
```

Размещение:

1. Найти на главной странице блок для владельцев.
2. Разместить изображение в визуальной колонке блока.
3. На desktop использовать двухколоночную композицию: текст и CTA в одной колонке, постер — в другой.
4. На tablet сохранить две колонки, если ширина текста остаётся читаемой; иначе перевести в одну колонку.
5. На mobile 390px выводить изображение отдельным блоком после заголовка и поясняющего текста, перед CTA либо перед списком преимуществ.
6. Не использовать изображение как декоративный фон под длинным текстом.

Настройки изображения:

```text
Формат: WebP
Исходный размер: 1200 × 900
Соотношение сторон: 4:3
object-fit: cover
object-position: center
Скругление: 12–16px
Тень: лёгкая либо отсутствует
Lazy load: включить
```

Не добавлять текст поверх изображения. Не изменять цветокоррекцию без необходимости.

## Что нельзя менять

- Не менять структуру сайта и URL без подтверждения.
- Не менять header/footer menu без подтверждения.
- Не добавлять Elementor.
- Не добавлять ACF как обязательный компонент.
- Не использовать неподтвержденные факты.
- Не ломать mobile 390px.
- Не использовать этот постер в hero.
- Не размещать на изображении встроенный текст, CTA или логотип.

## Критерии приемки

- [ ] Постер отображается именно в блоке владельцев на `/`.
- [ ] На desktop изображение не растянуто и не искажено.
- [ ] На tablet композиция не создаёт слишком узкую текстовую колонку.
- [ ] На mobile 390px нет горизонтального скролла.
- [ ] Основной объект изображения не обрезан бессмысленно.
- [ ] Alt-текст заполнен.
- [ ] Lazy loading включён.
- [ ] Не сломаны соседние блоки.
- [ ] Сохранена концепция A: navy / warm / green CTA / gold hover.
- [ ] Opencode указал, что именно изменено.
- [ ] Пользователь подтвердил визуально, если автоматическая проверка ограничена.
