
## Источник замечания

```text
ChatGPT Web / Gate 2 v2 / design system / reviewer instructions
```

## Проблема

Для страницы или шаблона **«Вторичная недвижимость»** ранее был подготовлен технический SVG-visual. Он не соответствует ожиданию по качеству итогового hero/poster-изображения для production. Требуется использовать **готовый растровый постер WebP** в стиле главной страницы.

## Где

```text
URL/шаблон: /buyers/catalog/resale/
Блок: hero и ключевые смысловые секции
Viewport: desktop 1440px, tablet 768px, mobile 390px
```

## Основание анализа

- Структура URL и обязательные верхнеуровневые блоки взяты из `gate2-v2.md`.
- Цвета, типографика, карточки, risk UX и требования к изображениям взяты из design system.
- Пользователь явно уточнил, что для остальных страниц нужны **постеры, а не многослойные SVG**.
- Перед изменениями OpenCode должен открыть текущую страницу, зафиксировать существующие блоки и не удалять рабочие функции без необходимости.

## Назначение страницы

выдача реальных объектов вторичного рынка либо индивидуальный подбор.

- Hero
- Фильтры
- Карточки объектов либо empty state
- Risk badges
- Сравнение
- CTA

- `Подобрать квартиру`

## Медиафайлы

Использовать **только production poster**:

```text
беклог-промтов/assets/site-pages/poster-livingroom-consultation-v1.webp
```

Роль постера:

- подбор, показы, консультация, вторичный рынок, отзывы;
- premium editorial visual в стилистике главной страницы;
- без текста, логотипов и интерфейсных элементов;
- poster visual, а не фотография конкретного объекта.

Alt:

```text
Вторичная недвижимость: выдача реальных объектов вторичного рынка либо индивидуальный подбор
```

Если изображение используется только как декоративное и смысл полностью передан соседним HTML-текстом, допустим пустой `alt=""`.

## Размещение и адаптивность

1. На desktop использовать hero 2 колонки: смысл/CTA слева, visual справа.
2. Контейнер страницы: 1180–1296px; hero не растягивать бесконтрольно на 1920px.
3. На tablet 768px допускается 55/45 либо переход в вертикальный stack.
4. На mobile 390px: текст → CTA → изображение; одна колонка, поля страницы около 20px.
5. Изображение: `aspect-ratio: 16 / 9`, `object-fit: cover`, focal point — центр/право.
6. Скругление: 18–24px; тень только лёгкая card/elevated из design system.
7. Не встраивать заголовок или кнопку внутрь SVG/WebP.
8. Lazy load включить, кроме визуала над fold, для которого использовать приоритетную загрузку без блокировки LCP.
9. Проверить отсутствие горизонтального скролла на 390px.

## Что нельзя менять

- Не менять URL и структуру меню без подтверждения.
- Не добавлять Elementor.
- Не добавлять ACF как обязательный компонент.
- Не удалять работающие формы и инструменты без фиксации причины.
- Не использовать AI-visual как фотографию конкретного объекта.
- Не добавлять неподтверждённые цифры, отзывы, партнёров, офисы, гарантии или результаты.
- Не использовать AI-изображения как фото реальных квартир.
- Не показывать фиктивные объявления.
- Risk badge всегда сопровождать текстом.

## Формы и состояния

Если на странице есть форма или инструмент:

- persistent labels, placeholder не заменяет label;
- поле не ниже 48–52px;
- success, empty и error states;
- следующий шаг после результата;
- consent/privacy checkbox или связанный legal text;
- дисклеймер с областью применения и ограничениями;
- на mobile primary action — во всю ширину.
