done

## Проблема

Главная страница должна использовать полноценный архитектурный hero-постер. Ранее в пакет были включены многослойные SVG, которые не являются корректным production-постером и удалены из актуального архива.

## Где

```text
URL: /
Блок: Hero / первый экран
Viewport: desktop 1440–1920px; tablet 768px; mobile 390px
```

## Как сейчас

Перед изменением проверить фактический production. Не использовать удалённые SVG-файлы из предыдущих версий пакета.

## Как должно быть

Использовать два готовых растровых постера:

```text
беклог-промтов/assets/home-hero-desktop-v2.webp
беклог-промтов/assets/home-hero-mobile-v2.webp
```

Desktop:

- использовать `home-hero-desktop-v2.webp`;
- изображение занимает правую часть первого экрана либо отдельную visual-колонку;
- слева остаётся чистая зона под H1, value proposition и CTA;
- ориентировочная высота hero 560–680px;
- использовать `object-fit: cover`;
- не искажать пропорции изображения.

Mobile:

- использовать отдельный `home-hero-mobile-v2.webp`;
- не делать агрессивный автоматический кроп desktop-файла;
- изображение располагать после основного текста и CTA либо в позиции responsive-макета;
- ориентировочная высота 220–320px на viewport 390px;
- не допускать горизонтального скролла.

Рекомендуемый alt:

```text
Современный жилой комплекс — экспертные решения для продажи и покупки недвижимости
```

## Что можно менять

- [ ] настройки Kadence Customizer;
- [x] Gutenberg/Kadence Blocks;
- [x] CSS;
- [ ] шаблон темы;
- [x] контент страницы;
- [x] медиа/ассеты;
- [x] PHP/WP-CLI скрипт, который обновляет главную страницу.

## Что нельзя менять

- Не менять структуру сайта и URL без подтверждения.
- Не менять header/footer menu без подтверждения.
- Не добавлять Elementor.
- Не добавлять ACF как обязательный компонент.
- Не использовать неподтверждённые факты.
- Не ломать mobile 390px.
- Не использовать старые SVG-постеры.
- Не добавлять текст, кнопки или логотип непосредственно в изображение.
- Не использовать этот poster как фотографию конкретного объекта каталога.

## Критерии приемки

- [ ] Desktop использует `home-hero-desktop-v2.webp`.
- [ ] Mobile использует `home-hero-mobile-v2.webp` через `<picture>`, media query либо отдельную настройку блока.
- [ ] В hero отсутствуют ссылки на удалённые SVG-постеры.
- [ ] H1 и CTA читаются без наложения на сложную часть изображения.
- [ ] Исправление проверено на 1440px, 768px и 390px.
- [ ] Нет горизонтального скролла на mobile 390px.
- [ ] Не сломаны соседние блоки.
- [ ] Сохранена концепция A: navy / warm / green CTA / gold hover.
- [ ] WebP-файлы отдаются с корректным MIME type `image/webp`.
- [ ] Opencode указал, что именно изменено.
