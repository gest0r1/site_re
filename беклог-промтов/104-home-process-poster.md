# 104 — Добавить P1-постер процесса работы на главную

## Статус

```text
new
```

## Источник замечания

```text
ChatGPT Web / проектный дизайн / макет главной
```

## Проблема

На главной странице блок «Как проходит работа» / процесс сопровождения не имеет отдельного предметного визуала, поддерживающего концепцию «Экспертность + доверие».

## Где

```text
URL: /
Блок: «Как проходит работа» / процесс сопровождения
Viewport: desktop 1440px, tablet 768px, mobile 390px
Устройство/браузер, если важно: все актуальные браузеры
```

## Как сейчас

Визуальный акцент блока отсутствует либо недостаточно выражен. Отдельного тематического постера для этого сценария нет.

Вывод основан на проектной документации и целевом макете. Точное текущее состояние продакшена перед изменением проверить отдельно.

## Как должно быть

Использовать файл:

```text
беклог-промтов/assets/home-process-poster-v1.webp
```

Назначение изображения:

- этапы работы: анализ, стратегия, реализация и сопровождение;
- спокойный премиальный real-estate визуал;
- без текста, логотипов и интерфейсных элементов внутри изображения.

Рекомендуемый alt:

```text
Планирование сделки и сопровождение недвижимости
```

Размещение:

1. Найти на главной странице блок «Как проходит работа» / процесс сопровождения.
2. Разместить изображение как основной visual этого блока.
3. На desktop использовать изображение в отдельной визуальной области/колонке или как крупный supporting poster внутри блока.
4. На tablet сохранить читаемую композицию и не допускать слишком узкой текстовой колонки.
5. На mobile 390px выводить изображение отдельным блоком до или после основного текста — выбрать вариант, который не ломает последовательность чтения.
6. Не использовать изображение как декоративный фон под длинным абзацем.

Настройки изображения:

```text
Формат: WebP
Исходный размер: 1400 × 800
Соотношение сторон: 7:4
object-fit: cover
object-position: center
Скругление: 12–16px
Тень: лёгкая либо отсутствует
Lazy load: включить
```

Не добавлять текст поверх изображения. Не изменять цветокоррекцию без необходимости.

- Не ломать mobile 390px.
- Не использовать как фон под длинный текст; это supporting visual для блока процесса.
- Не размещать на изображении встроенный текст, CTA или логотип.

## Критерии приемки

- [ ] Постер отображается именно в целевом блоке на `/`.
- [ ] На desktop изображение не растянуто и не искажено.
- [ ] На tablet композиция не создаёт слишком узкую текстовую колонку.
- [ ] На mobile 390px нет горизонтального скролла.
- [ ] Основной объект изображения не обрезан бессмысленно.
- [ ] Alt-текст заполнен.
- [ ] Lazy loading включён.
- [ ] Не сломаны соседние блоки.
- [ ] Сохранена концепция A: navy / warm / green CTA / gold hover.
- [ ] Opencode указал, что именно изменено.
- [ ] Пользователь подтвердил визуально, если автоматическая проверка ограничена.
